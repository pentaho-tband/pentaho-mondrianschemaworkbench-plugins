The Mondrian Schema Workbench

Introduction

The Mondrian Schema Workbench is a designer interface that allows you to create and test Mondrian OLAP cube schemas visually. Pentaho Analysis is built on the popular Mondrian OLAP engine. The Pentaho BI Platform embeds the Mondrian engine as part of its architecture. Along with the Pentaho BI Platform, the Mondrian engine processes MDX requests with the ROLAP (Relational OLAP) schemas. These schema files are XML metadata models that are created in a specific structure used by the Mondrian engine. These XML models can be considered cube-like structures which utilize existing FACT and DIMENSION tables found in your RDBMS. It does not require that an actual physical cube is built or maintained; only that the metadata model is created. The structures and related API are outlined in the Mondrian Technical Guide that accompanies the Mondrian distribution as well as the Schema Workbench.

Note: See also, Publishing an Analysis Schema Using the Schema Workbench and Creating a New Analysis View.

The Mondrian Schema workbench provides you with the following functionality:

  • A schema editor integrated with the underlying data source for validation (see above)
  • The ability to test MDX queries against schema and database
  • The ability to browse the underlying database's structure

Starting the Schema Workbench

To start the Schema Workbench:

At the command prompt, type:

In Windows:

ant workbench-run,

or

java -jar lib/workbench.jar

In UNIX/Linux:

java -jar lib/workbench.jar

Go to Tools > Preferences to set properties for connecting to your cube database.

These preferences and the last four schemas edited are kept in a workbench.properties file in the root of the classpath. If you change the connection properties, you must close/save the existing schema editor(s) and restart to see the effects.

Creating and Opening a Schema

If you open an existing schema, Schema Workbench validates that the tables and columns underlying the cube definitions actually exist in the database.

The Schema Workbench validates edits against the Mondrian schema DTD.

Save your schema and run some MDX queries to test. Error messages and results are displayed.
